Programmation asynchrone


La programmation asynchrone est un concept clé en JavaScript, car de nombreuses opérations, comme les requêtes réseau ou les opérations d'entrée/sortie, peuvent prendre un certain temps à se terminer. JavaScript utilise un modèle d'exécution asynchrone pour éviter de bloquer le fil d'exécution principal pendant que ces opérations sont en cours.

Callbacks

Un callback est une fonction passée en argument à une autre fonction, qui sera appelée lorsque l'opération asynchrone sera terminée. C'était la première approche pour gérer le code asynchrone en JavaScript.


// Fonction asynchrone simulée
function fetchData(callback) {
  setTimeout(function() {
    const data = { name: 'John', age: 30 };
    callback(null, data);
  }, 2000);
}

// Utilisation du callback
fetchData(function(err, data) {
  if (err) {
    console.error(err);
  } else {
    console.log('Données récupérées :', data);
  }
});

Bien que les callbacks soient fonctionnels, ils peuvent rapidement devenir difficiles à gérer lorsqu'ils sont imbriqués (problème de "callback hell").



Promesses (Promise)

Les promesses sont une abstraction plus récente et plus élégante pour gérer le code asynchrone. Une promesse représente une valeur potentielle qui peut être résolue (resolve) ou rejetée (reject) à un moment futur.


// Fonction asynchrone simulée retournant une promesse
function fetchData() {
  return new Promise(function(resolve, reject) {
    setTimeout(function() {
      const data = { name: 'John', age: 30 };
      resolve(data);
    }, 2000);
  });
}

// Utilisation de la promesse
fetchData()
  .then(function(data) {
    console.log('Données récupérées :', data);
  })
  .catch(function(err) {
    console.error(err);
  });

Les promesses peuvent être chaînées et facilitent la gestion des opérations asynchrones.


Async/await


Introduite avec ES2017, la syntaxe async/await offre une manière plus lisible et plus intuitive d'écrire du code asynchrone en utilisant les promesses.


// Fonction asynchrone simulée retournant une promesse
function fetchData() {
  return new Promise(function(resolve, reject) {
    setTimeout(function() {
      const data = { name: 'John', age: 30 };
      resolve(data);
    }, 2000);
  });
}

// Utilisation de async/await
async function getData() {
  try {
    const data = await fetchData();
    console.log('Données récupérées :', data);
  } catch (err) {
    console.error(err);
  }
}

getData();


La syntaxe async/await permet de traiter les promesses de manière plus linéaire et plus séquentielle, ce qui rend le code plus lisible et plus facile à maintenir.


La programmation asynchrone est essentielle pour créer des applications web performantes et réactives. Que vous utilisiez des callbacks, des promesses ou async/await, comprendre ces concepts est crucial pour gérer correctement les opérations asynchrones en JavaScript.
